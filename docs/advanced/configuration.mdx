# Configuration

ShipMobile stores all project-specific configuration in a `.shipmobile/` directory at your project root.

## Directory Structure

```
.shipmobile/
├── credentials.json      # Encrypted account credentials
├── config.json           # Project configuration
├── metadata/             # Generated store metadata
│   ├── ios.json
│   └── android.json
└── audit-history/
    └── latest.json       # Last audit result (for --diff)
```

## credentials.json

Stores your connected accounts:

- **Expo/EAS** — Access token, username, validation timestamp
- **Apple Developer** — API Key ID, Issuer ID, path to `.p8` key, team name, validation timestamp
- **Google Play** — Path to service account JSON, validation timestamp

Credentials are encrypted at rest. Each entry includes a `validatedAt` timestamp — if older than 30 days, `shipmobile doctor` will flag it.

## config.json

Project-level settings created by `shipmobile init`:

- Detected project type
- Bundle IDs
- Target platforms
- Build preferences

## metadata/

Generated by `shipmobile prepare`. Contains app name, description, keywords, category, and privacy policy templates — ready for store submission.

## audit-history/

Saved by `shipmobile audit`. The `latest.json` file is used when running `shipmobile audit --diff` to compare scores over time.

## .gitignore

Add `.shipmobile/` to your `.gitignore`:

```gitignore
.shipmobile/
```

<Warning>
  Never commit `.shipmobile/credentials.json` to version control. It contains your account credentials.
</Warning>

## Resetting

To clear everything and start over:

```bash
shipmobile reset --force
```

This deletes the entire `.shipmobile/` directory.
