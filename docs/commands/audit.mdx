# shipmobile audit

Deep static analysis of your project for store readiness. Produces a 0â€“100 score with actionable findings.

```bash
shipmobile audit
```

## Options

| Flag | Description | Default |
|------|-------------|---------|
| `-p, --path <path>` | Project path | `.` |
| `-c, --category <cat>` | Run specific category only | All |
| `--fix` | Auto-fix issues where possible | `false` |
| `--diff` | Compare with previous audit | `false` |

Categories: `performance`, `memory`, `ux`, `compliance`, `security`

### Performance (5 rules)

| Rule | Description |
|------|-------------|
| `bundle-size` | Checks overall bundle/dependency weight |
| `heavy-deps` | Flags heavy dependencies that bloat the app |
| `unused-imports` | Detects unused import statements |
| `large-images` | Finds oversized image assets |
| `hermes-check` | Verifies Hermes engine is enabled |

## Auto-Fix

```bash
shipmobile audit --fix
```

Rules with a `fix` function will automatically remediate issues (e.g., removing `console.log` statements). The audit re-runs after fixes to give you an updated score.

## Diff Mode

```bash
shipmobile audit --diff
```

Compares the current audit against the last saved result in `.shipmobile/audit-history/latest.json`. Shows score delta and new/resolved findings.

<Tip>
  Run `shipmobile audit --category security` before every commit to catch leaked secrets early.
</Tip>
